# コマ1 演習：単体テスト入門（自販機）

Java + JUnit 5 の単体テストを「写経」と「穴埋め」で体験します。
コードをゼロから書く必要はありません。

## 使い方

1. VSCode でこのフォルダを開く（拡張機能「Extension Pack for Java」を入れておく）
2. ターミナルで `mvn test` を実行する
3. 左サイドバーの「フラスコ（試験管）」アイコンから、テストを1本ずつ実行もできる

## 演習の順番

### 演習①-2：写経して、1本動かす（10分）
`src/test/java/vending/CalcTest.java` を開き、コメントの手順どおりに
テストを **手で打って** みる（コピペ禁止）。`mvn test` で緑（成功）を見る。

### 演習①-3：穴埋めで、仕様書をコードにする（20分）
`src/test/java/vending/VendingMachineTest.java` を開く。
`_____` の部分に、自分が設計したテストケース仕様書の値を入れる。

- ケース1：完成例（まず Green を確認）
- ケース2：正常系（お釣り）← 穴埋め
- ケース5・6：境界値（140円 / 150円）← 穴埋め（true / false）
- 異常系（金額不足・売切れ）は **設計まで**。コードはコマ2で書きます。

## 困ったとき集

| 症状 | だいたいの原因 |
|------|----------------|
| 赤い波線が出る | セミコロン `;` の付け忘れ、カッコ `)` `}` の閉じ忘れ |
| `expected: <50> but was: <0>` | 左が「正しい答え（期待値）」、右が「今の結果」。期待値を見直す |
| 緑の ▷（実行ボタン）が見当たらない | 拡張機能「Extension Pack for Java」が入っているか確認 |
| `cannot find symbol` | クラス名・メソッド名のスペルミス（大文字小文字も区別される） |

## ファイルの場所

```
    CalcTest.java            （演習①-2：写経）
    VendingMachineTest.java  （演習①-3：穴埋め）
```
